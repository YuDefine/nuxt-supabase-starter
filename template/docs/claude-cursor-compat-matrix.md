# `.claude` -> `.cursor` 能力對照表（初版）

> 範圍：`template/.claude` 與 `template/.cursor`  
> 目的：判斷是否能達成 Cursor 100% 等價能力，並標記缺口

## 狀態標記

- `等價`：Cursor 可原生達到同等效果
- `可替代`：可達近似效果，但觸發條件或行為語意不同
- `需重寫`：需要改寫流程或包裝層，不能直接沿用
- `不可等價`：平台能力模型不同，無法 1:1 複製

## 1) 核心設定層

| `.claude` 能力                                   | 目前 Cursor 對應                  | 狀態     | 說明                                                                                 |
| ------------------------------------------------ | --------------------------------- | -------- | ------------------------------------------------------------------------------------ |
| `.claude/settings.json` hooks 生命週期           | `.cursor/hooks.json`              | 可替代   | 已完成事件映射（`Pre/PostToolUse`, `SessionStart`, `Stop`）。仍需 matcher 實測校正。 |
| `permissions.allow` 白名單（Bash/MCP/Skill/Web） | 無 1:1 對應                       | 不可等價 | Cursor 無同格式全域 allow-list；需改為 hooks/流程治理。                              |
| `enabledMcpjsonServers`                          | `.mcp.json` + Cursor MCP 啟用機制 | 需重寫   | 設定來源與載入時機不同，需逐服務驗證。                                               |
| `includeGitInstructions`                         | 無直接對應                        | 不可等價 | Cursor 無同名行為開關。                                                              |

## 7) 總結判定

| 面向             | 判定                                                   |
| ---------------- | ------------------------------------------------------ |
| 是否可 100% 等價 | 否                                                     |
| 是否可高覆蓋運作 | 是                                                     |
| 目前估計覆蓋率   | 約 80%~90%（取決於 hook matcher 與 MCP 啟用狀態）      |
| 最大缺口         | `permissions.allow`、skills 原生承接、MCP 載入語意差異 |

## 8) 優先補強順序

1. **Hook matcher 實測校正**：先驗證 `Skill`、`MCP`、`Shell` 三類 matcher 是否確實命中。
2. **MCP 啟用一致化**：讓 `local-supabase` 在 Cursor 端可穩定調用。
3. **Skills 橋接清單化**：先選 20 個高頻 skills 做明確映射（命令/agent/hook）。
4. **權限治理替代方案**：用 `beforeShellExecution` / `beforeMCPExecution` hooks 重建 allow/deny 策略。
